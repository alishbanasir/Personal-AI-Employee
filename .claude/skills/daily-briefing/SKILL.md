# Daily Briefing

Generate a structured daily briefing and write it to `/Briefings/YYYY-MM-DD_Briefing.md`.

## Step 1 — Gather Data

Read the following files:
1. `Business_Goals.md` — targets and active projects
2. `Dashboard.md` — current state
3. `/Done/` folder — completed tasks (filter to relevant date range)
4. `/Logs/YYYY-MM-DD.json` — today's action log (if exists)
5. `/Needs_Action/` — what's still pending

## Step 2 — Write Briefing

Create `/Briefings/<TODAY>_Briefing.md` using this template:

```markdown
---
generated: <ISO timestamp>
period: <date range>
---

# Daily Briefing — <TODAY>

## Executive Summary
<2-3 sentences summarizing the day>

## Completed Today
<List tasks moved to /Done/ today with checkboxes>

## Still Pending
<List items in /Needs_Action/ with priority and age>

## Awaiting Approval
<List items in /Pending_Approval/ with action type and age>

## Bottlenecks
<Identify any items older than 24 hours that are stuck>

## Recommendations
<1-3 proactive suggestions based on vault state>

---
*Generated by AI Employee v0.1*
```

## Step 3 — Update Dashboard

After writing the briefing:
1. Add a "Briefing generated" entry to Dashboard.md Recent Activity
2. Link to the briefing file: `[Today's Briefing](Briefings/<filename>)`

## Rules

- Be concise — briefings should be readable in under 2 minutes
- Highlight anything overdue or stuck
- Always include a "Recommendations" section with at least one suggestion
- Save to `/Briefings/` not the vault root
